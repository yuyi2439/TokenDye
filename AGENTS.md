# TokenDye — Agent Guide

## Project structure

- `tokendye/` — core library (`DyeModule`, `DyeLabel`, `ModelDyeConfig`, `DyeDataset`).
  Published as wheel in `dist/`. Entrypoint is `tokendye/__init__.py`.
- `run_Qwen3.5-4B/` and `run_Qwen2.5-7B-Instruct/` — model-specific training/test scripts.
  Each has `train.py`, `test_model.py`, `ablation_test.py`, `utils.py`, `DyeConfig.json`.
  Model weights live in subdirs (`Qwen3.5-4B/`, `Qwen2.5-7B-Instruct/`) — gitignored or submodules.
- `dataset/` — git submodule (`DyeDataset.git`). Contains `.jsonl5` data files, generator, classifier.
- `log_viewer.py` — plots loss/gradient norms from training log files.

## Setup

- Python >= 3.12, managed via `uv`.
- Core deps: `pydantic`, `jsonl5>=0.1.1`. Runtime deps: `transformers`, `accelerate`, `bitsandbytes`, `matplotlib`.
- Optional GPU kernels (install separately): `flash-attn`, `flash-linear-attention`, `causal_conv1d`.
- Models load with 4-bit quantization (BitsAndBytes `nf4`) and `flash_attention_2`.

## Commands

```bash
# Train (runs from inside a model-specific directory)
python train.py

# Dataset generation (requires ~/.api_key with OpenRouter key)
python dataset/generator.py --rounds N

# Add dedup data to existing dataset
python dataset/add_data.py

# Sanity-check dataloader
SANITY_CHECK=1 python train.py

# Ablation test (uses latest workspace auto-detection for Qwen2.5-7B,
# requires WORKSPACE env var for Qwen3.5-4B)
python ablation_test.py

# LR sweep (Qwen3.5-4B only)
python sweep_lr.py

# Log viewer
python log_viewer.py <output_dir>

# Build library wheel
uv build
```

## Architecture notes

- Training freezes base model (`model.requires_grad_(False)`). Only `DyeModule` params train.
- A `DyeModule` is a LoRA-like `nn.Linear(d_model, rank)` → `nn.Linear(rank, d_model)` pair.
- One `DyeModule` per label. Modules registered in `nn.ModuleDict`, optimized jointly.
- Applied via `forward_hook` on `model.model.embed_tokens`. The hook reads `_dye_mask` from the embedding layer at each forward pass.
- During `generate()`, the hook **skips** self-generated tokens (dye_mask length mismatch with output sequence length) — only input tokens get dyed.
- Resuming training (`RESUME_TRAINING`) is stubbed with `raise Exception("Todo")`.
- Checkpoints save `dye_modules_best.pt` (dye weights only) and `train_state_best.pt` (full optimizer state).

## Dataset format

JSONL5 (JSON5 line-delimited). Each line:
```json5
{segments: [{dye: "system", text: "..."}, {dye: "user", text: "..."}], target: "..."}
```

- `target` is optional (inference-only entries omit it).
- Dye labels are defined in `DyeConfig.json` per model.
- `dataset/add_data.py` reads from `.newdata.json5` and deduplicates against existing data.
- `dataset/classify_data.py` categorizes entries by injection type.

## Test / Verify

- No test framework; verification is manual.
- `test_model.py` runs a simple generation with sampling params.
- Use `SANITY_CHECK=1` to verify dataloader output before full training.
- `log_viewer.py` plots train/val loss and per-label gradient norms from training logs.

## Conventions

- Output dirs: `<run_dir>/.outputs/<timestamp>/`
- Log files: `<output_dir>/train-<case>.log` (DE-BUG level for gradient norms, INFO for epoch stats)
- Dataset paths: `run_Qwen2.5-7B-Instruct/utils.py` hardcodes `./dataset/v0.1a.jsonl5`; `run_Qwen3.5-4B/train.py` hardcodes `./dataset/v0.2.jsonl5`.
- `DyeConfig.json` is model-specific and generated by running `utils.py` standalone.
- `d_model` must match `model.config.hidden_size`; `dtype` must match model compute dtype.
